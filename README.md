Flight Booking System

## About the Project
Flight Booking System is a full-stack web application that allows users to browse flights, book tickets, and manage reservations. The project demonstrates frontend-backend communication through a REST API, database integration with SQLite, and deployment to a cloud hosting platform.

## Technologies 
- Fetch API — sending requests to the server (frontend communicates with backend)
- Node.js — backend runtime environment (runs JavaScript outside the browser)
- Express.js — web framework for building APIs and backend services
- server.js — main backend server file (handles API and database operations)
- app.js — frontend application logic (client-side functionality)
- REST API — communication between frontend and backend through HTTP requests
- SQLite — lightweight relational database stored in a single `.db` file
- SQL — querying and managing database data
- localStorage — storing user settings, preferences, and theme locally
- GET Requests — retrieving data from the server
- POST Requests — sending new data to the server
- Async/Await — asynchronous programming for API interactions
- Caching — temporary storage of data for faster access and performance
- API Routes — backend endpoints responsible for different functionalities
- JSON — data exchange format between frontend and backend
- CRUD Operations — Create, Read, Update, and Delete data operations
- Git — version control system
- GitHub — source code hosting and collaboration platform
- Render — cloud deployment and hosting platform
